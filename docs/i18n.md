# Internationalisierung (i18n)

## Überblick

Das Projekt nutzt **[vue-i18n](https://vue-i18n.intlify.dev/)** (v11) für die Mehrsprachigkeit. Aktuell werden zwei Sprachen unterstützt:

| Locale | Sprache | Status |
|---|---|---|
| `de` | Deutsch | 🟢 Standard-Locale |
| `en` | Englisch | 🟢 Fallback-Locale |

---

## Setup

Die i18n-Instanz wird in `src/locales/index.ts` konfiguriert:

```typescript
import { createI18n } from 'vue-i18n';
import de from './de.json';
import en from './en.json';

const i18n = createI18n({
  legacy: false,          // Composition API Modus
  locale: 'de',           // Standard-Sprache
  fallbackLocale: 'en',   // Fallback bei fehlendem Key
  messages: { de, en },
});

export default i18n;
```

**Wichtig:** `legacy: false` aktiviert den Composition-API-Modus. Dadurch wird `useI18n()` in `<script setup>` verwendet.

---

## Verwendung in Komponenten

### In `<script setup>`

```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n';

const { t } = useI18n();
</script>

<template>
  <h1>{{ t('feature.login.headline') }}</h1>
</template>
```

### Mit Interpolation

```vue
<!-- Template -->
<p>{{ t('feature.recipes.createRecipe.ingredients.servingsText', { servingChooser: 4 }) }}</p>

<!-- Ergebnis DE: "Das Rezept ist für 4 Personen" -->
<!-- Ergebnis EN: "The recipe is for 4 people" -->
```

### In Route-Meta

Routen definieren einen `titleKey`, der in `TitleFrame.vue` automatisch aufgelöst wird:

```typescript
{
  path: '/recipes',
  component: RecipesView,
  meta: { titleKey: 'feature.recipes.title' },
}
```

## Dateistruktur der Übersetzungen

Die JSON-Dateien folgen einer hierarchischen Struktur:

```json
{
  "navigation": {
    "home": "Startseite",
    "plan": "Plan",
    "recipes": "Rezepte",
    "browse": "Entdecken"
  },
  "feature": {
    "<featureName>": {
      "title": "...",
      ...
    }
  }
}
```

### Konventionen

- **Top-Level-Keys:** `navigation` für Navigationseinträge, `feature` für Feature-Texte
- **Feature-Namensraum:** `feature.<featureName>.*`
- **Verschachtelung:** Maximal 4–5 Ebenen
- **Key-Sprache:** Englisch (kebab/camelCase)

---

## Neue Sprache hinzufügen

1. **JSON-Datei** erstellen: `src/locales/<locale>.json`  
   Kopiere die Struktur von `de.json` oder `en.json` und übersetze alle Werte.

2. **Import** hinzufügen in `src/locales/index.ts`:
   ```typescript
   import fr from './fr.json';
   
   const i18n = createI18n({
     // ...existing config...
     messages: { de, en, fr },
   });
   ```

3. **Fertig!** Die neue Sprache kann über `i18n.global.locale.value = 'fr'` aktiviert werden.

---

## Neue Übersetzungsschlüssel hinzufügen

1. Schlüssel in **allen** Locale-Dateien (`de.json`, `en.json`, …) eintragen
2. Im Code über `t('pfad.zum.schlüssel')` verwenden
3. Wenn ein Key nur in einer Locale fehlt, greift automatisch der `fallbackLocale` (`en`)
